# 【YouTube Trending Collector】規格計劃書

## 1. 專案概述

### 1.1 專案背景與目的

YouTube 創作者最缺的從來不是「剪輯技巧」，而是「題材」——每天看一堆國外影片找靈感，但手動整理太慢，也無法系統性地分析「什麼樣的標題最有效」。現有的 YouTube 分析工具多半是給廣告主用的，關心的是 CPM 和觸及人數，而非「這個影片的標題公式我能不能抄」。本工具專注於「創作者視角的內容靈感引擎」：自動監控指定頻道或關鍵字的熱門影片，AI 分析標題公式、預測病毒潛力、並產出可操作的下一步建議。目標是讓創作者每天打開工具，10 分鐘內就能得到「今天值得做的 5 個題目」。
### 1.2 目標受眾（TA）

- YouTube 創作者：需要持續找到新題目，但靈感來源散落各處
- 內容策劃：需要系統性追蹤競爭對手的內容策略
- 社群行銷團隊：需要知道什麼內容在 YouTube 最火，跨平台調整
- MCN / 經紀公司：需要監控旗下創作者的表現與市場趨勢
### 1.3 專案範圍

- 頻道 URL 輸入監控、關鍵字監控、類別追蹤（遊戲/科技/美食等）
- 熱門影片抓取（觀看次數/讚數/留言排序）、影片元數據（標題/描述/標籤）
- 頻道訂閱數變化、AI 熱門話題萃取、標題公式分析
- 病毒潛力評分（1-10）、內容建議、每日/每週摘要報告
- Discord / Email 推播、歷史資料查詢
- YouTube 影片下載（版權問題）、實際上傳或排程發布
- 留言分析與管理、頻道收益估算

## 3. 視覺與 UI

### 3.1 品牌設計指南

- Primary #FF0000：YouTube 品牌紅、主要按鈕
- Secondary #0F172A：深色背景、主要容器
- Accent #10B981：成功狀態、高病毒潛力
- Warning #F59E0B：中等病毒潛力｜Danger #EF4444：低病毒潛力
- Background #0B0F19 / Card BG #1F2937
- 字體：標題 Inter 700、內文 Inter 400/500、數字 JetBrains Mono
### 3.2 跨裝置支援

- Desktop ≥ 1024px：完整功能，儀表板多欄，影片分析側邊欄
- Tablet 768-1023px：單欄，影片卡片 Grid 2 列
- Mobile < 768px：精華體驗，儀表板摘要、每日摘要閱讀、歷史查詢
## 4. 前端功能規格

- 儀表板：今日 TOP 10、監控概覽、AI 建議精華
- 監控設定：新增頻道 URL / 關鍵字，設定類別與更新頻率
- 熱門影片列表：依監控頻道/關鍵字分類，多維度排序
- 影片卡片：縮圖/標題/頻道名/觀看數/讚數/留言數/日期
- 影片分析頁：基本資訊 + AI 標題公式 + 病毒潛力（1-10）+ 相關題目建議
- AI 內容建議：「基於這個影片，你可以做...」的可操作建議
- 每日摘要報告：AI 生成，含今日熱門話題 TOP 3、標題公式觀察
- 報告推播：Discord Webhook / Email 每日定時推送
- 頻道成長折線圖（30天）、熱門標題排行榜 TOP 20
- 歷史資料查詢：日期範圍 + 關鍵字查詢
## 5. 後端與技術規格

### 5.1 技術棧

- 前端框架：Next.js 14（App Router）+ Tailwind CSS
- YouTube 資料：YouTube Data API v3（官方）或 yt-dlp（備案）
- AI 分析：Claude API 或 GPT-4o
- 後端框架：FastAPI + PostgreSQL
- 排程：Celery + Redis
- 部署：Railway（API + Worker）+ Vercel（前端）
### 5.2 第三方 API

- YouTube Data API v3：影片/頻道資料查詢（免費 10,000 單位/天 quota）
- yt-dlp：備用爬蟲（官方 API 失敗時，開源免費）
- Claude API：AI 標題分析與內容建議（按 token 計費）
- Discord Webhook：每日摘要推播（免費）
- SendGrid：Email 每日摘要（可選，免費 100 封/天）
